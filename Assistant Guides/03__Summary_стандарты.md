# 03 — Summary стандарты (папки и большие файлы)

Обновлено: 2026-01-11

Цель: ассистент находит нужное **по якорям**, а не читает всё подряд.

## 1) Summary папки — что внутри (обязательно)
Файл: `00__SUMMARY.md` внутри папки.

Структура:
1) Где мы: путь (human).
2) Назначение папки (2–6 строк).
3) Полный список “на уровень вниз”:
   - подпапки: 1–2 строки
   - файлы: 1–3 строки
4) Поисковые якоря.
5) Что читать первым (entry points).

> [!tip]- Raw-ссылки в summary
> Для ассистента лучше, если у каждого пункта есть `raw:` ссылка на файл/summary.

## 2) Размер summary
- цель: 300–1200 слов
- потолок: ~12 KB текста
Если вылезли — дробим на подпапки и делаем новые summary ниже.

## 3) Summary внутри большого файла (опционально)
Рекомендуемый формат — foldable callout:
```md
> [!summary]- Summary
> - 3–10 буллетов: что внутри
> - ключевые NPC/локации/хуки
> - см. также: соседние файлы
```

Технические примечания отдельно:
```md
> [!info]- Технические заметки
> - правила/оговорки/TODO
```

## 4) Обновление summary (реверсивно вверх)
Если файл изменён/добавлен/удалён:
- обновляем summary папки,
- при “значимых” изменениях обновляем summary родителя (и выше при необходимости).
