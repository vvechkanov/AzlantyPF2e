# 02 — Структура и сущности проекта

Обновлено: 2026-01-11

Этот гайд фиксирует “что за файлы вообще существуют” и какую роль они играют.

## Сущности
- **AZL__00_ACCESS.md** (Project Files): одна точка входа, ссылка на GitHub index.
- **AZL_ASSISTANT_INDEX.md** (GitHub): bootloader для ассистента.
- **00__SUMMARY.md** (GitHub/Obsidian): summary для папки (корня или любой ветки).
- **Контентные файлы**: `.md/.json` с каноном, сценами, NPC и т.д.
- **(Опционально) codes map**: если используем кодирование путей (старый AZL-режим).
- **(Опционально) manifest**: если используем delete/move как операции.

## Главное правило
Summary — это “карта”:
- отражает **всё, что есть на уровень вниз** (подпапки + файлы),
- даёт якоря и “куда идти”,
- содержит raw-ссылки на перечисленные элементы (в идеале).

## Почему мы разбиваем гайды
Один большой документ тяжело поддерживать.
Модульные гайды:
- проще обновлять;
- проще читать (точечно);
- легче держать “точки входа” короткими.
