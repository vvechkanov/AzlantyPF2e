# 01 — Доступ и навигация в GitHub (для ассистента)
Обновлено: 2026-01-12

GitHub используется **только для ассистента** как read-only источник текста.  
Obsidian — основное место для человека.

## Что читать первым
1) `AZL__00_ACCESS.md` (в Project Files) — даёт ссылку на ROOT SUMMARY.
2) Root `00__SUMMARY.md` (в GitHub) — единая точка входа и карта веток.
3) Дальше: summary-дерево → `00__SUMMARY.md` нужной папки → целевой файл.

## RAW-ссылки
Ассистент читает **raw**, а не `blob`:
- ✅ `https://raw.githubusercontent.com/.../.md`
- ❌ `https://github.com/.../blob/.../.md`

> [!important] Про токены
> Не использовать ссылки вида `...?token=...`. Для публичного чтения токены не нужны.

## Полное чтение файлов (чтобы избежать обрезаний)
Иногда просмотрщик может показать только фрагмент файла.  
Если нужен полный контекст — **скачай RAW целиком и работай по локальной копии** (в песочнице).

Правило:
- перед серьёзной правкой/выжимкой смысла из файла → скачать raw → читать локально целиком → править.

## Root Summary vs Summary папки — в двух строках
- **Root `00__SUMMARY.md`** — единая точка входа: правила + переходы к главным веткам.
- **`00__SUMMARY.md` папки** — карта конкретной ветки: подпапки/файлы “на уровень вниз” с RAW-ссылками.

## Как ассистент “ходит по дереву”
1) Открыть root summary → перейти в нужную ветку.
2) Открыть summary ветки → открыть целевой файл по raw-ссылке.

## Glossary (глоссарий ассистента)
Глоссарий — это “словарь ссылок”: **термин/имя → куда идти читать канон**.

- Файл: `Glossary/00__GLOSSARY.md` → raw: https://raw.githubusercontent.com/vvechkanov/AzlantyPF2e/master/Glossary/00__GLOSSARY.md

Как пользоваться:
1) Увидел непонятный термин / имя NPC/PC / механику / локацию / фракцию.
2) Открыл глоссарий.
3) Нашёл запись (включая алиасы).
4) Перешёл по RAW на каноничную страницу.

Как пополнять:
- если в тексте появился новый значимый термин или NPC/PC → обновить глоссарий;
- для NPC/PC: запись глоссария должна вести на их страницу.

## Если в Summary нет ссылок
Цель проекта — чтобы summary содержали raw-ссылки на перечисленные файлы.
Если ссылки нет:
1) Не гадать и не “восстанавливать по памяти”.
2) Добавить raw-ссылки в `00__SUMMARY.md` этой папки.
3) Если `00__SUMMARY.md` папки отсутствует — создать его (редкий, но допустимый случай).
