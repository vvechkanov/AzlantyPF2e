# 04 — Рабочий процесс и бандлы

Обновлено: 2026-01-11

Этот гайд описывает “как переносить изменения” между ассистентом и Obsidian.

## Текущее решение
Основная база знаний хранится в GitHub и читается ассистентом по raw-ссылкам.

ZIP-бандлы используем, когда:
- ассистент генерирует/правит много файлов локально для Obsidian,
- или нужно передать пакет изменений “одним действием”.

## Правило выдачи
Ассистент **не выдаёт** файлы/zip автоматически.

Бандл отдаётся только по явной команде:
- «Собери бандл»
- «Дай AZL__BUNDLE»

## Что должно быть в бандле (если используем)
- изменённые `.md/.json`
- затронутые `00__SUMMARY.md`
- (опционально) `AZL__MANIFEST.json` если есть delete/move операции

## Важно: кодировка имён в ZIP (кириллица)
Иногда при упаковке/распаковке ZIP появляются “кракозябры” в названиях папок (например, вместо `Сюжет кампании`).

Чтобы этого не было:

### Как создавать бандл
- Имя архива всегда строго: `AZL__BUNDLE.zip` (без суффиксов).
- Создавай архив **через 7-Zip** или **через Python (zipfile)**.
- **Не** создавай архив встроенным Проводником Windows (“Отправить → Сжатая ZIP-папка”) — он часто ломает имена на кириллице.

### Быстрая проверка
- Открой `AZL__BUNDLE.zip` в 7-Zip и убедись, что пути внутри читаются нормально (кириллица без кракозябр).
- Если внутри уже кракозябры — архив нужно пересоздать другим способом (7-Zip / Python), иначе распаковка создаст “мусорные” папки.

### Правило совместимости
- Пути внутри архива должны **точно** совпадать со структурой папок Obsidian/репозитория (включая кириллицу и пробелы).
