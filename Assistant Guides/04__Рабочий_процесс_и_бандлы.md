# 04 — Рабочий процесс и бандлы
Обновлено: 2026-01-12

Этот гайд — для ассистента: как оформлять изменения и как собирать `AZL__BUNDLE.zip`.

## Базовый принцип
Предпочитать **редактирование существующих файлов**.  
Новые файлы создавать только по необходимости (новая сущность или отсутствующий обязательный summary).

## Шаг 0 — всегда читать файл целиком
Если файл важен для канона/структуры — **скачать RAW целиком и работать по локальной копии**, чтобы не упереться в обрезание фрагментов.

## Обязательные шаги после генерации/правок
1) Обновить/создать каноничную страницу сущности (сначала правка, потом новый файл).
2) Обновить затронутые `00__SUMMARY.md` (включая summary арок/подпапок).
3) Если появились новые значимые термины/имена/фракции/локации/механики — обновить `Glossary/00__GLOSSARY.md`.
4) Для NPC/PC:
   - обновить их AI INDEX (FILES и LINKS), если появилось новое упоминание или новая связь.

## Изображения NPC
Изображения хранятся во Foundry.  
В NPC-файлах указывается прямая ссылка на изображение (или `raw: TBA`, если ещё нет ссылки).

## Бандл (ZIP)
- Бандл создаётся только по явной просьбе пользователя.
- Имя: `AZL__BUNDLE.zip`
- Внутри: только новые/изменённые файлы, с реальной структурой папок.
- Delete/move/rename — только через `AZL__MANIFEST.json` (version=1, ops=[...]).
