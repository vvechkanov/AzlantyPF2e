# 08 — Кодировки и переводы строк (UTF‑8 / LF) для Win + Linux + GitHub

Обновлено: 2026-01-22

Цель: чтобы файлы **одинаково читались** в Obsidian на Windows/Linux и **корректно отображались** на GitHub (без «кракозябр» и без случайных CRLF).

## Инварианты (обязательные правила)
- **Кодировка**: все текстовые файлы проекта храним в **UTF‑8 без BOM**.
- **Переводы строк**: в репозитории используем **LF**.
- **Запрещено**: сохранять файлы как cp1251/ANSI (на GitHub это часто превращается в «кракозябры»).

## Что в репозитории это обеспечивает
- `.gitattributes` фиксирует `eol=lf`, чтобы Git приводил переводы строк предсказуемо.
- `.editorconfig` просит редакторы сохранять `charset=utf-8`, `end_of_line=lf`, `insert_final_newline=true`.

## Практика на Windows (как “применится” локально)
- Если используешь редактор с поддержкой EditorConfig (VS Code / JetBrains / Notepad++ с плагином) — он автоматически будет соблюдать правила.
- Git при checkout будет держать LF для файлов, на которые распространяется `.gitattributes`.

## Для ассистента (важно)
- При любых правках не вводить BOM, не переводить файлы в cp1251/ANSI, не ломать LF.
- Если видишь `No newline at end of file` — добавить финальную новую строку.
